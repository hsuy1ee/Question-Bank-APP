# Question Bank Practice

A local Windows desktop app for question-bank practice. It supports importing JSONL question banks and provides sequential practice, random practice, wrong-question practice, and favorite-question practice.

## Features

- Import `.jsonl` question banks
- Support single-choice, multiple-choice, and true/false questions
- Support sequential practice, random practice, wrong-question practice, and favorite-question practice
- Track answer history, accuracy, wrong-question count, and favorite count
- Convert Word question banks to JSONL when the Word file follows the expected format
- Store question banks and study records in a local SQLite database

## Build Guide

### Requirements

- Windows
- Node.js 24 or newer
- npm

Install dependencies:

```bash
npm install
```

### Development

Start the Electron development version:

```bash
npm run dev
```

### Build

Run TypeScript checks, compile the Electron main process, and build the frontend:

```bash
npm run build
```

This command generates:

```text
dist/
dist-electron/
```

### Package Windows Installer

Create the Windows installer:

```bash
npm run dist
```

The packaged output is written to:

```text
release/
```

Common files include:

```text
题库刷题 Setup 0.1.0.exe
题库刷题 Setup 0.1.0.exe.blockmap
win-unpacked/
```

The `.exe` file is the installer. `win-unpacked/` is the unpacked portable app directory. `.blockmap` is generated by electron-builder and is mainly used for differential downloads in auto-update workflows. If you install the app manually, you can ignore the `.blockmap` file.

## Question Bank Format

The import format is JSONL: one question per line, and each line is a complete JSON object.

Single-choice example:

```json
{"id":"q001","type":"single","question":"Question text","options":["Option A","Option B","Option C","Option D"],"answer":["A"],"explanation":"Answer: A (Option A)","tags":["single"],"source":"Sample question bank"}
```

Multiple-choice example:

```json
{"id":"q002","type":"multiple","question":"Which of the following statements are correct?","options":["Option A","Option B","Option C","Option D"],"answer":["A","C"],"explanation":"Answer: AC","tags":["multiple"],"source":"Sample question bank"}
```

True/false example:

```json
{"id":"q003","type":"judge","question":"True/false question text","answer":["true"],"explanation":"Answer: A (A, true)","tags":["judge"],"source":"Sample question bank"}
```

Field descriptions:

- `id`: Unique question ID
- `type`: Question type. Allowed values are `single`, `multiple`, and `judge`
- `question`: Question text
- `options`: Options. Required for single-choice and multiple-choice questions
- `answer`: Correct answer, always stored as a string array
- `explanation`: Answer or explanation
- `tags`: Tags
- `source`: Question bank source

## Word to JSONL

The project includes a Word-to-JSONL conversion script:

```text
scripts/convert-docx-to-jsonl.ps1
```

Example:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\convert-docx-to-jsonl.ps1 `
  -InputDocx ".\your-question-bank.docx" `
  -OutputJsonl ".\converted\your-question-bank.jsonl"
```

The script expects the Word question bank to use a format like this:

```text
1. [单选] Question text
A. Option A
B. Option B
C. Option C
D. Option D
答案：A（Option A）
```

```text
2. [多选] Question text
A. Option A
B. Option B
C. Option C
D. Option D
答案：ABCD
```

```text
3. [判断] Question text
A. A、是
B. B、否
答案：B（B、否）
```

If the Word question format changes significantly, the conversion script must be updated accordingly.

## Local Data

The app uses `sql.js` to store a local SQLite database. You do not need to install MySQL, PostgreSQL, SQLite server, or any separate database service.

Default database location:

```text
C:\Users\<your-username>\AppData\Roaming\question-bank-desktop\question-bank.sqlite
```

The database stores:

- Imported question banks
- Questions
- Answer records
- Wrong-question records
- Favorite records
- Dashboard statistics

Deleting the project directory does not automatically delete this database.
